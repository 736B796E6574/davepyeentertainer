HTML Structure

In this section we will present you the main HTML structure of all files.

As you know Musician comes with three layout options. Every layout has specific header, content and footer. Although the content is different the main HTML structure is the same for every layout. We will explain all parts of HTML structure later, now we will show you the basic setting of it.

So what you need to know? As any HTML file we have head and body tag. Inside of the head tag we have some meta information about the template, like title, description, author and tags. This is important for SEO purposes. Every page of your website should have unique title and description. Also, inside of the head tag we have calls for stylesheets, template css files and Google web fonts.

Ok now we know what is in the head tag. Let's see what is in body tag. There is your content and at the bottom there is some script tags, used for calling specific jQuery plugins and scripts. Those are needed for specific elements on the page to function, which we will talk about a bit later.
So how is the content organized. Inside of the body we first have header. Header contains logo, main navigation and search. As this is HTML template search is not functional by default. But you can use custom Google search if you need that functionality.
Next thing we have in the body tag is content, actual content of your site. Every peace of content is placed inside a div with class "page-content". Page content can have background color or background image. You can read more about page-content settings in page content settings section . Inside of page content we have classical Bootstrap structure. That is container, div and the columns. Inside the columns you can place content elements. You can have more than one page content on a page. Consider it a section of content. At the end we have footer containing some widgets and copyright info. So this is basic HTML structure of the template.

Working with Sass

In this section we will explain you the benefits of sass, and how can you benefit it in Musician.

If you never worked with Sass, please first read this article . it explains basic idea of using sass and how to use it. So first to use Sass you need to have some code editor and install sass on your computer. There is a lot of nice tutorials on how to do that :). Of course it depends on code editor you are using. We are using NetBeans. You can see how to install Sass in NetBeans here.

So let's see why we used Sass. As templates are more and more complicated and have gazillion of options, what you get in the end is a pretty large product, with many features and code you might not need. As we want that you get a lightweight website we used Sass, where we divided all template parts into separate files. So you can include only the css files you actually need.

Ok let's go further. In your download folder you will see scss folder. This is a folder containing all sass files. If you installed Sass on your computer and set it up in your code editor we are good to go. Let's just explain in a few sentences how this work. You have a folder scss containing sass files. On your website you will not use sass files, but standard css files.

So why you need sass for? As we said before, sass files are compiled by your code editor. So everything you do or change in style.sass file, or any other sass file will be compiled into your main style.css file. So what is the benefit?

In Musician we have a lot of elements. So every element is placed inside one "partial" file. That means that inside main style.sass file there will be import for all those files, or for the files you actually need. What this means? If you don't use call to action element on your website, you can open style.sass file and delete "import elements-call-to-action" part. This means that in style.css file you use on your website, there won't be styling for call to action. So you will have less code and smaller file size, your website will load faster and it's easier to maintain. Nice, right?

Ok now you have an idea how this works. We have many small, partial sass files, which are used to style specific thing. We can import those files into main sass file. That main sass file will generate our standard style.css file which we will use on our template/your website. Now, let's see which files are those and how to work with them.
Please open your scss folder. There are files that are named like "_something.scss". This "_" means that this is partial file (file that we import into main file). Also there are two main sass files - those ones will generate our css files. Let' see everyone of them.

  • _variables.scss

    If you read the article we mentioned earlier, you need that in Sass there are some things called variables. We define a variable, asign a value to it and then we use that variable across all the file. For example, we have a variable called "$mc-light". This means that main-color-light has value of "#aaaaaa". So if we change that value in variables.scss file, we have changed it across all document. The same thing is for base font-family. If you need to change font-family, simply edit variable that says "$body-font-family". And it will change across all document. pretty nice, right?

  • _mixins.scss

    What are mixins? Those are parts of code that makes your job easier and faster. For example we have a mixin called "@mixin body-font-stack()". In this mixin we used variables that tell what is the basic font family, size, color etc. So now, when editing an element and we need to give it all those options, we don't need to write them all, we simply import a mixin. And the mixin itself will add all those options we deifined.

  • _base.scss

    Ok now let's see other files. All of those files uses variables and mixins we defined before. this particular file, as the name says is used for basic styling. That is body, typography, lists, etc.

  • _layout_base.scss

    This file is used for styling the basic layout of the template. So there we have styling for page-content, backgrounds for page content, sidebar, page titles and there we have some helper classes, like mb-0, which means "margin-bottom: 0;". We will discuss more about helper classes later. So if you want to change page-content backgrounds, paddings or margins, here is the place to do that.

  • _layout-headers.scss

    As the file name says, this is file that has all the styling for headers of the template. Inside you will find general styling for all header types, top bars, main navigation, all that is placed inside a header on this template. of course you can easily remove the stuff you don't need. Every part of the file is commented so it will be easy to change what you need.

  • _layout-footer.scss

    This file contains all the footer styling.

  • _elements-*.scss

    Now we have couple of "elements-" files. As you can see, every element this template offers is placed into separate file. In that way you can quickly find which element you want to change and if you want to include it in your main style.css file. how to do that? That's where main style.scss file comes into play.

  • style.scss

    As you can see this file doesn't contain any styling. It is used only for calling other sass files. but when your code editor compiles it, it will make style.css file, a standard css file that will contain all the styling that is present in those partial files you imported in your main sass file. So as you can see we have "import" statement in that file after which we have a list of files. All of these files will be compiled and placed into one single file called style.css. So, here is the main reason why we used sass. If you don't need accordion, tabs or tables on your website, then simply open style.scss file and delete call for that elements. You will have only the code you need in your main style.css file. This will make it easier to maintain, it will be smaller size, which influences on your page load. So this is a nice way to have a lightweight website.

We hope we taught you something about benefits of Sass and that you will use it to customize this template and every future template you buy in the future, because we believe it will make your job a lot easier and faster, and most important the end product will be of higher quality.

General Layout Customization

In this section we will explain the general layout of the template and show you how to choose the color style, set up the header and footer, etc.

Choosing a layout

As you can see in the demo and your download files, Musician has three layouts, music artist, music band and music blog. The main difference between each of them is the content of course, header, footer and elements used. You can use any of the element that is used in this template and any of the header and footer you like. This means if you like music band layout but would like to use header from music blog you can. You simply need to copy/paste header you like. After you decide which layout you would like to use, you can remove any unnecessary HTML files.

You can choose between dark layout(music artist and music band) and music blog, white background layout. If you want to use dark theme just add "dark-theme" class to the body tag. If you want to use white theme, remove that class.

Music artist uses only "dark-theme" body class. If you want music band layout styling just add "music-band" class next to "dark-theme". If you prefer music blog layout add simply add class "music-blog". Music blog is white layout so it doesn't have "dark-theme" class.

Choosing color style

Every layout has specific color style. Each color style is placed into separate css file. You can find it in css folder. Color files are named "color-*.css". You can edit css file that is used in the layout, so you don't need to replace the file in each HTML file. So for example music artist layout uses color-default css file which is blue. You can simply open that css file and replace the green values with the color of your brand. The new color will be applied on all pages in music artist layout. The same goes for all other layouts.

Understanding the grid

As we already stated, Musician is based on Bootstrap 3. This means that it uses Bootstrap's 1170px grid with 12 columns. Each column is placed inside the row. Row is placed inside a container, which can be fixed width - 1170px or fluid - takes whole page width. To use fluid container simply use class "container-fluid". Container is placed inside page-content. That is the main structure you must follow.

Elements comes inside a column. You can combine columns in a way that sum of their numbers is 12. So for example you can have two col-md-6 in one row. or you can have three col-md-4 in one row.

You can have more than one row inside a container. Also, you can have more than one page content inside a page. Page content is just a section of content on which you can add background image or color. So if you have a part of a page that has some custom background, place it in a page-content and give that page content a class that defines it's background. More about page content element in the next section.

You can read more about Bootstrap grid and how you can combine and use columns on their official pages.

You can see example of grid on elements grid HTML page to see typical columns combination and difference between full-width and fixed container width.

Page content backgrounds

As you can see in the demo files, some sections - page-content divs have image or color background. This is defined in style.css file so you can make customizations there. If you use Sass, you can make customizations in "_layout_base.scss" file.

So how do you know that page-content has a custom background? We have custom classes for that. First of all we will have class "custom-background". That class defines custom padding on that section. After that we can have a class that defines a background image or color. For example class "bkg-img02" defines that page-content will have image background which is defined by that class in style.css file. Also, when we have a page-content that has some type of custom background we need to say that it needs to have bottom margin, so that the content that comes after doesn't stick to it. We have couple of helpers classes for that, but we always used (in this template), class "mb-120", which says margin-bottom is 120px.

In music band layout on page contents with background image we added decoration border with class "bkg-border". This class adds :after and :before svg image at the top and bottom of page content. You can change that image with another one in img/svg folder.

You can read more about custom, helpers classes in helper classes sections.

Full width and sidebar pages

As you can see in Musician there is option to use full width pages and pages with sidebar. You can use left and right sidebar.

When you use page with sidebar, then the whole content is placed inside one page-content. Content of the page is then placed inside "col-md-9" and sidebar is placed inside "col-md-3".

if you want to have page with left sidebar, then in your HTML first place the code for sidebar. It will be col-md-3 with class aside-left. After that place the col-md-9 which will hold the content.

If you want to use the page with right sidebar, then first place your page content and after that the sidebar. You can see examples in demo files.

Also as you can see there are some "widgets", that is elements in the sidebar. You can place any element in the sidebar, you just need to leave the main structure the same.

How to make your site retina ready

This is pretty simple. On Musician we used svg format for every icon, and font icons, so every icon will look sharp on each screen. So you don't have to do anything here.

What you need to do is to make your photos retina ready. So each photo you use, make one that is twice bigger. Here you have some naming standard that you will have to use, because of the retina.js script that will handle the switch of images on retina screens. Yes, script will do all the job needed in the back. So what you need to do is following. let's say you have image that is called image01.jpg. This image is 400x600px. make the same image in double size, that is 800x1200 and name it image01@2x. This part is important. Your double size image needs to have that @2x part. How to generate an Instagram Access Token

There are few easy steps to generate your own Instagram token in a safe and secure manner.

STEP 1

Head over to http://instagram.com/developer. Login to your account. Then click on Manage Clients, and finally on Register a New Client

STEP 2

Fill out the required fields for registering a new client id making sure to set the OAuth redirect_uri field to "http://localhost" (without quotes).

The reason you should use localhost is for security. In computer networking localhost is the web address of the computer you are using. So when Instagram redirects to your redirect_uri, all of your sensitive information is passed from Instagram's servers directly to your computer. See wikipedia.org/wiki/Localhost for more details.

Also, be sure to uncheck the "Disable implicit OAuth" box or else this method will not work.

Once all of the fields are filled out click Register.





STEP 3

Paste the following url into your web browser.

add the client id you generated to the url replacing "[CLIENT_ID_HERE]" and also add you redirect uri instead of "[REDIRECT_URI_HERE]" (without quotes).

Hit enter, and you will connect to Instagram's servers. Instagram will ask you whether you want to grant the client you registered in Step 2 access to your account. Once your client has been granted access to your account, Instagram will redirect your browser to your localhost and your newly generated access token will be appended to the url after localhost/#access_token=
